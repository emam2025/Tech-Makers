## 1. الصفحة الرئيسية `/`

**الهدف:** تحويل الزائر لمسجل

### الأقسام
1. **Hero** — خلفية gradient أزرق، شعار "الدفعة الخامسة 2026"، عنوان رئيسي "من مستهلك للتكنولوجيا إلى صانع..."، فئات عمرية 8-11 / 12-15 / 16-20، زرين CTA
2. **الإحصائيات** — 4 بطاقات (الطلاب, المسارات, الجلسات, سنوات الخبرة) — حالياً hardcoded
3. **المسارات** — 3 بطاقات (Track A/B/C) مع أيقونات
4. **الطلاب المتميزين** — Grid صور الطلاب (11 طالب)
5. **آراء العملاء** — بطاقات تقييم
6. **الأسئلة الشائعة** — Accordion
7. **دعوة للتسجيل** — CTA section

**الصور:** `hero-bg.jpg`, `techmakers-opt.jpg`, `technomath-opt.jpg`, `secondary-banner.jpg`, `/student/1.png`-`11.png`, `hero.jpeg`, `join-banner.jpg`, `tracks-banner.jpg`

**API:** لا يحتاج (كل البيانات hardcoded حالياً)

## 2. صفحة المسارات `/tracks`

**الهدف:** عرض تفصيلي للمسارات

### الأقسام
1. **Hero** — صورة خلفية + عنوان
2. **بطاقات المسارات (3)** — أيقونة، وصف، زر "اعرف المزيد"
3. **المستويات** — لكل مسار: السنوات، الموضوعات، الأدوات
4. **الأسعار** — 3 خطط (Basic/Pro/Enterprise)
5. **الأسئلة الشائعة** — Accordion (5-6 أسئلة)
6. **CTA** — زر "سجل الآن"

**ملاحظة:** هذا الملف 86KB — الأكبر في المشروع، يحتاج تقسيم

## 3. تسجيل الطلاب `/register`

**الهدف:** نموذج تسجيل متعدد الخطوات

### الخطوات
1. **الخطة والمسار** — اختيار نوع الخطة (شهري/نص سنوي/سنوي) + المسار
2. **البيانات الشخصية** — الاسم، تاريخ الميلاد، الجنس، الرقم القومي، الهاتف
3. **البيانات الأكاديمية** — المدرسة، الصف الدراسي، المحافظة
4. **معلومات التواصل** — البريد، هاتف ولي الأمر، العنوان، ملاحظات

**API:** `POST /api/register` — يرسل لـ Supabase + Brevo (بريد إلكتروني)

---

## 4. الانضمام للفريق `/join`

**الهدف:** نموذج تقديم للمدربين

### الخطوات
1. **معلومات شخصية** — الاسم، البريد، الهاتف، الدولة (16 دولة hardcoded)
2. **المؤهلات** — الدرجة العلمية، الجامعة، التخصص، رفع الشهادات
3. **الخبرات** — سنوات الخبرة، الخبرات السابقة، التخصصات (8 مجالات)
4. **اللغات والصورة** — اللغات، رفع الصورة الشخصية

**API:** `POST /api/join` — يرسل لـ Supabase

## 5. التحقق من الشهادات `/certificate`

**الهدف:** بحث عن الشهادة برقم قومي أو هاتف

### العناصر
1. **نموذج البحث** — اختيار نوع البحث + قيمة البحث + زر بحث
2. **بطاقة النتيجة** — اسم الطالب، المسار، المستوى، تاريخ الإصدار، رقم الشهادة
3. **مودال المشاركة** — واتساب (16 دولة) + إيميل

**API:** `GET /api/certificate` — يستخدم Google Apps Script Proxy

## 6. اختبار اللغة `/english-test`

**الهدف:** اختبار تحديد مستوى تفاعلي (30 سؤال)

### المراحل
1. **شاشة البداية** — وصف الاختبار، زر "ابدأ الاختبار"
2. **الاختبار** — عداد الأسئلة، شريط التقدم، السؤال + 4 اختيارات
3. **النتيجة** — النسبة المئوية، مستوى CEFR (A1-C2)، تفصيل المهارات

**API:** `POST /api/english-test` (Gemini AI لتوليد وتحليل الأسئلة)

**6 تبويبات:**

## العناصر المشتركة

### الهيدر (الموقع العام)
- Logo (يتغير حسب الثيم)
- روابط: الرئيسية، من نحن، برامجنا (قائمة منسدلة: تك ميكرز، تكنوماث، لغة إنجليزية، تحول رقمي، ثانوية عامة)، انضم إلينا، الشهادات
- زر Dark/Light toggle
- Responsive: قائمة Hamburger للموبايل

### الشريط الجانبي (Admin)
- Logo
- روابط Dashboard + كل وحدات الإدارة
- زر طي/توسيع
- Responsive: يتحول لـ Drawer من اليسار في الموبايل

### DataTable (مكون مشترك)
- أعمدة قابلة للتخصيص
- فرز
- بحث
- Pagination
- أزرار إجراءات (تعديل/حذف/عرض)
- Empty State: "لا توجد بيانات"

## ملاحظات للمصمم

1. **الجمهور المستهدف:** أطفال (8-20) + أولياء أمور — يحتاج تصميم جذاب ملون مع عناصر مرحة
2. **RTL:** الموقع بالكامل عربي (direction: rtl)
3. **الألوان الحالية:** أزرق ملكي `#2F6FE4`، ذهبي `#F59E0B`، أخضر `#22C55E`، أحمر `#E04B5A`
4. **الخطوط:** IBM Plex Sans Arabic (العربية)، Poppins (الإنجليزية)
5. **التجاوب:** Mobile-first مع breakpoints: sm(640), md(768), lg(1024), xl(1280)
6. **الحركات:** Framer Motion (fade-in, slide-up, stagger children)
7. **الأيقونات:** Google Material Symbols
8. **الصور:** مجلد `/public/` يحتوي على ~2.6MB من الصور المحسّنة

### الأولويات
- **عالية:** إعادة تصميم الصفحة الرئيسية + Dashboard admin
- **متوسطة:** صفحات المسارات + التسجيل + اختبار اللغة
- **منخفضة:** باقي صفحات admin (تتبع نفس النمط)

### احتياجات حالية
1. إضافة "نسيت كلمة المرور" في صفحة login
2. Empty states لكل الجداول
3. Skeleton loading للـ Dashboard
4. Error boundaries لكل صفحة
5. إزالة الـ hardcoded data من Dashboard
